# Avoid Abstract Rules

## 禁止文言

以下のような抽象的な指示は AGENTS.md / CLAUDE.md / Cursor rules に書いてはいけない。  
トークンを消費するだけで、AIの行動を変えることがないため。

```
- 常に最高品質で実装する
- 必ず深く考える
- すべての観点でレビューする
- 完璧なコードを書く
- ベストプラクティスに従う
- セキュリティを考慮する
```

## 具体化の例

抽象的な指示を、具体的で確認可能な基準に置き換える。

### auth 変更時

```
auth変更時は以下を確認する:
- 権限境界が正しいか（ロール/スコープが意図通りか）
- 未認証アクセスが適切にブロックされるか
- 横断アクセス（他ユーザーのリソースへのアクセス）が防止されているか
- ログ出力に認証イベントが含まれるか
```

### DB migration 変更時

```
DB migration変更時は以下を確認する:
- rollback が可能か
- 既存データに影響があるか
- nullable / default の設定は妥当か
- index が適切に追加されているか
- lock time が許容範囲か（テーブルサイズが大きい場合は特に注意）
```

### UI 変更時

```
UI変更時は以下のステートを確認する:
- empty state（データがないとき）
- loading state（読み込み中）
- error state（エラー発生時）
- mobile / responsive behavior
```

### API 変更時

```
API変更時は以下を確認する:
- リクエスト/レスポンスの型が変更されたか
- 破壊的変更（breaking change）かどうか
- 既存クライアントへの影響
- API docs/OpenAPI spec が更新されているか
```

### インフラ変更時

```
インフラ変更時は以下を確認する:
- 既存環境への影響（既存リソースの削除や変更がないか）
- ロールバック手順があるか
- コスト変動の見積もり
- 本番環境ではなくステージングで先に検証可能か
```

## 原則

ルールを追加するときは、必ず以下を満たすこと。

1. 「何を確認するか」が具体的に書かれている
2. AIが確認結果を yes/no で判断できる
3. 確認しなかった場合に起きる具体的な失敗がある
