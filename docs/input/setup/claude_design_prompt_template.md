# claude_design コンテキストテンプレート

まあみが claude_design でUI案を生成するときに、AIパートナーが参照してclaudie_designに送るテンプレートです。
**まあみはこのファイルを直接操作しない。** AIパートナーが `ls components/` で既存コンポーネント一覧を動的に収集して充填した上でclaudie_designに送る。

---

## 使い方（まあみ → AIパートナーへの依頼）

まあみはAIパートナーに一言伝えるだけ:

```
「〇〇コンポーネントのUI案を2つ生成して」
```

AIパートナーが `ls components/` でコンポーネント一覧を取得し、このテンプレートに充填して送る。
生成後は [DESIGN_FLOW.md](../../harness/DESIGN_FLOW.md) の Step 2（Gate 1チェック）へ進む。

---

## テンプレートの手動更新が必要なケース（AIパートナーが対応）

「既存コンポーネント一覧」以外の情報が古くなった場合のみAIパートナーが更新する:

| 変更内容 | 更新箇所 |
|---|---|
| カラートークン・デザインシステムが確定した | 「ビジュアルトーン」に追記 |
| 禁止語が追加された | 「禁止語」セクションを更新 |
| 技術スタックが変わった | 「技術スタック」セクションを更新 |

更新後は `/start-task` → コミット → PRを出すこと。

---

## テンプレート本文

```
---以下をコピーしてclaudie_designに貼る---

# COCOSiL V2 UI生成コンテキスト

## プロダクト概要
「自分を知って、ラクになる。」
対人関係で繰り返し消耗する25-35歳の若手社会人に、
AIと4体系パーソナリティ分析で「なぜそうなるか」を届けるWebサービス。

## 絶対に守るUXシーケンス（順序を変えない）
1. 共感 — ユーザーの悩みに共鳴する
2. 安心 — ここは安全だと感じさせる
3. 分析 — 診断結果・傾向を提示する
4. 行動 — 次のアクションを促す

⚠️ 分析結果を先に見せてはいけない。共感・安心が先。

## 禁止語（UIコピーに使わない）
- 占い / 占い師 → 性格分析 / パーソナリティ診断
- 鑑定 / 鑑定士 → 診断 / 分析
- 運勢 / 運命 → 傾向 / 特性
- 当たる（精度の意味）→ 一致度が高い / 精度が高い
- 霊感 / 霊視 → （使用不可）

## ビジュアルトーン & デザイントークン

### ブランドアイデンティティ
- ターゲット: 25-35歳 若手社会人（職場・恋愛・家族の人間関係に悩む層）
- 「根拠のある性格分析」であり、スピリチュアルサービスではない
- 落ち着いた・誠実・科学的な印象を優先する
- 急かす・チャラい・怪しい印象は避ける

### ブランドカラーパレット
| トークン | hex | Tailwindクラス | 用途 |
|---|---|---|---|
| primary | #1E293B | slate-800 | 見出し・重要テキスト・塗りボタン |
| primary-light | #334155 | slate-700 | サブ見出し・ホバー状態 |
| accent | #6366F1 | indigo-500 | リンク・バッジ・CTA以外の強調 |
| surface-1 | #F8FAFC | slate-50 | ページ背景 |
| surface-2 | #F1F5F9 | slate-100 | カード背景 |
| text-primary | #0F172A | slate-900 | 本文テキスト |
| text-secondary | #64748B | slate-500 | サブテキスト・補足文 |
| border | #E2E8F0 | slate-200 | 区切り線・カード枠 |

### UXフェーズ別カラー（COCOSiL固有・必ず守る）
コンポーネントを配置するUXフェーズに応じて色を選択する。

| フェーズ | hex | Tailwindクラス | 心理的役割 |
|---|---|---|---|
| 共感（Empathy） | #F59E0B | amber-500 | 暖かさ・寄り添い |
| 安心（Safety） | #F8FAFC | slate-50 | 静けさ・安全 |
| 分析（Analysis） | #475569 | slate-600 | 落ち着き・論理的根拠 |
| 行動（Action） | #F97316 | orange-500 | 前進・行動喚起 |

⚠️ フェーズをまたいだ色の流用禁止（例：共感フェーズのカードにslate-600を使わない）

### 状態色
| 用途 | hex | Tailwindクラス |
|---|---|---|
| 成功・完了 | #0D9488 | teal-600 |
| エラー | #DC2626 | red-600 |
| 警告 | #D97706 | amber-600 |

### ⚠️ カラー禁止ルール
- ❌ `gray-*` / `zinc-*` / `stone-*` → ニュートラルは `slate-*` のみ（混在でデザインが崩れる）
- ❌ `violet-*` / `purple-*` → スピリチュアル感が出るため禁止

### タイポグラフィ（Tailwindデフォルト＋逸脱ルールのみ）
- フォントウェイト: 見出しは `font-semibold`（`font-bold` は強すぎる）
- 行間: 本文は `leading-relaxed`（`leading-tight` は詰まって見える）
- 最小フォントサイズ: `text-xs`（それ以下禁止）

### スペーシング（8px grid基準）
- コンポーネント内パディング: `p-4`（16px）または `p-6`（24px）
- セクション間ギャップ: `gap-6`（24px）または `gap-8`（32px）
- カードボーダー半径: `rounded-xl`（他の値は使わない）

## 技術スタック（生成コードはこれに準拠すること）
- **フレームワーク:** Next.js 16 App Router（Server Components優先）
- **CSS:** Tailwind CSS **v4**（v3との互換はない）
  - ✅ `bg-black/50`（透明度は /スラッシュ表記）
  - ❌ `bg-opacity-50`（v3表記、使わない）
- **TypeScript:** strict mode
- **importパスエイリアス:** `@/*` = プロジェクトルート
  - ✅ `import { Button } from "@/components/Button"`
  - ❌ `import { Button } from "src/components/Button"`（`src/` は存在しない）
- **環境変数:** `@/lib/env` 経由のみ（`process.env.XXX` の直接参照禁止）
- **Zod:** `import { z } from 'zod/v4'`（サブパス必須）
- **認証:** Clerk（Clerk UIコンポーネントが既に導入済み。認証フローのUIは上書きしない）

## 既存コンポーネント一覧
（現時点では空。追加されたら都度このテンプレートを更新する）

## 今回の生成依頼
【コンポーネント名】: （例: DiagnosisResultCard）
【画面・機能の説明】: （例: MBTIタイプ診断結果を表示するカード。結果タイプ名・説明文・相性タイプへのリンクを含む）
【特記事項】: （例: 診断結果は「安心フェーズの後」に表示されるため、温かみと安心感を重視）

複数案（最低2案）を生成してください。
---ここまで---
```

## テンプレートの更新タイミング

以下が変わったらこのテンプレートを更新すること:

| 変更内容 | 更新箇所 |
|---|---|
| 新しいコンポーネントを `components/` に追加した | 「既存コンポーネント一覧」に追記 |
| カラートークン・デザインシステムが確定した | 「ビジュアルトーン」に追記 |
| 禁止語が追加された | 「禁止語」セクションを更新 |
| 技術スタックが変わった | 「技術スタック」セクションを更新 |

更新後は `/start-task` → コミット → PRを出すこと（このファイルも `docs/` 管理下）。
