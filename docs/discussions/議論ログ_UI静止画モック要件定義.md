# 議論ログ：UI静止画モック要件定義（Claude Designプロンプト化）

## 登場人物

- 🎨 **田中 優**（UI/UX × AI生成デザイン専門家）：モバイルUI設計歴10年。Claude Design / Figma AI での感情駆動型UI生成が専門。
- 🔬 **中村 真理**（プロダクトリサーチ × インタビュー設計専門家）：ユーザーリサーチ歴8年。インタビュー用プロトタイプ設計100件以上。
- 💅 **みさき**（ギャル）：IT素人だけど感性が鋭い。MBTI系診断アプリ大好きな実際のユーザー。

## Turn 1｜既存要件定義の診断 — Claude Designプロンプトとして何が足りないか

**🎨 田中：**
既存の `ui_mockup_requirements_interview.md` はレイアウト・要素・コピーは十分に定義されています。ただ、Claude Designプロンプトへの変換という観点で見ると、3つの欠落があります。

**①「空気感」の言語化が不足。**
「暗い紫グラデーション」と書いてあっても、それが「瞑想アプリの内省感」なのか「高級感のあるSPA感」なのかが不明です。Claude Designはこのトーンの違いで出力が大きく変わる。

**②コンポーネントの具体度が低い。**
「AIバブル・薄紫背景」とあるが、角丸半径・影・パディングが未指定です。Claude Designには「iMessageのような丸みのある吹き出し」などの参照イメージ語が必要。

**③感情的ヒエラルキーが視覚指示に落ちていない。**
要件定義は「何があるか」の一覧であって、「どれが主役でどれが脇役か」の優先度が視覚指示として定義されていない。

結論：この要件定義書は「Figmaで作る人向け」であり、「Claude Designのプロンプト向け」ではない。変換の核心は「機能名→ビジュアル感情言語への翻訳」です。

---

**🔬 中村：**
田中さんの分析に一点追加します。インタビュー用という観点で欠落しているのは、**「仮説を検証できる視覚的フック」の設計**です。

「デザインの美しさ」ではなく「感情反応の引き出し力」をプロンプトの目的変数に置く必要があります。Claude Designプロンプトには「インタビュアーが観察したい感情反応の種類」を組み込む必要があります。

---

**💅 みさき：**
私も診断系アプリいくつか使ったことあるんですけど〜。正直、結果画面ってどれも似てるんですよ。「あなたはINFJです」みたいな。

てかClaude Designって何ですか？…あー！「こんな画面を作って」って文章を書く作業ですね。難しそう〜。私だったら「なんかほっとする感じ」しか言えなさそう笑

## Turn 2｜Claude Designプロンプトの構造設計 — 何を、どの順序で書くか

**🎨 田中：**
みさきさんの「ほっとする感じ」は実は正解です。Claude Designプロンプトで最も重要なのは「感情語」を冒頭に置くことです。

私の実務経験から、UIプロンプトの有効構造は次の順序：

```
① 感情・空気感（2〜3語）
② モバイルスクリーン仕様（サイズ・背景色）
③ レイアウト構造（上から下への要素配置）
④ 差別化要素の強調（【最重要】ラベルで明示）
⑤ タイポグラフィ・カラー仕様
⑥ 【必須コピー】の固定（文言変更不可指示）
⑦ 禁止・排除事項
```

①を最初に置くと、AIが全体のトーンを先確立してから細部を作るため一貫性が出る。

COCOSiLの5画面の感情語：

| 画面 | 感情語 |
|------|--------|
| 画面1（LP） | 「期待・好奇心・信頼」 |
| 画面2（共感対話） | 「温かく自然な会話の空気」 |
| 画面3（安心フェーズ） | 「静寂と受容・深呼吸のような間」 |
| 画面4（統合レポート） | 「知的満足・腑落ち・自己認識」 |
| 画面5（チャット） | 「理解されている安心感・対話の温かさ」 |

---

**🔬 中村：**
インタビュー用という制約から、もう一層必要です。**「意図的に情報を間引く」指示**です。

情報が多すぎると、インタビュー相手が感情反応ではなく「機能説明を読む知的評価」モードに入ります。Claude Designへの追加指示：

> 「画面の主役は1〜2要素のみ。その他は視覚的に後退させる。細かいダミーテキストは最小化。」

これを指示しないと、Claude Designはリアルなアプリに見せようとして情報を詰め込みます。

---

**💅 みさき：**
「マジ当たってる！」って感動するのって、結果が詳しいからじゃなくて、「え、これ私のこと？」って一瞬なるやつですよね。キャッチーな一言が大きく出てるほうが刺さる気がする〜。めっちゃ小さい文字がいっぱいあっても全然読まないし笑

## Turn 3｜画面ごとの「見せ場」確定 — クリティカル1要素の選定

**🔬 中村：**
みさきさんの「一言に凝縮」感覚を元に、5画面それぞれの「クリティカル1要素」を確定させます。

| 画面 | クリティカル1要素 | なぜそれか |
|------|-----------------|-----------|
| LP | 「共感→安心→分析→行動」UXシーケンス図 | 「何のアプリか」の最速伝達手段 |
| 共感対話 | AIバブルコピー「今日、どんな気持ちで来ましたか？」 | この問いへの反応が仮説の核心 |
| 安心フェーズ | 「これは評価ではありません。地図です。」コピー | V2最大差別化。最重点観察点 |
| 統合レポート | 4体系バッジ横並び × 統合タイプ名 | 「4体系」独自性を一目で証明 |
| AIチャット | 診断バッジ + 診断名参照コピーの共存 | 「普通のAIと違う」証拠 |

Claude Designプロンプトではこのクリティカル1要素を【最重要】ラベルで明示し、他の要素は「後退させる」と明示する。

---

**🎨 田中：**
「後退させる」の具体的な書き方：

**正しいプロンプト例（画面3）：**
```
静寂と受容の空気感。深呼吸のような間。
モバイルUI、375×812px。
背景：#1E0B3B〜#5B21B6 暗い紫グラデーション（全画面）。

【最重要】呼吸サークル（白の同心円4重・波紋状・中央配置）。
         静止画でも「動いている」感を表現する。

【必須コピー】「これは評価ではありません。
              あなた自身を理解するための地図です。」白・中央。

「読んでみる →」半透明白ボタン・Thumb Zone内。
ボトムナビ：非表示。装飾・アイコン・余分なラベルは一切排除。
```

**NG例（情報詰め込み）：**
「ヘッダー、プログレスバー、アニメーション、タイトル、サブテキスト、ボタン、ボトムナビ、アイコン...」
→ 要素を並列列挙すると全要素が等価扱いになる。

---

**💅 みさき：**
あー「これは評価じゃない」って言葉、超刺さる気がします！診断系アプリ開くとき「当たってるかな〜？外れてたら恥ずかしいかな？」みたいな変な緊張感あるんですよ。それを先に解いてくれる感じ。

あと画面3って何も書いてないから「読み込み中？」って誤解されない？そこだけ心配で〜。

## Turn 4｜禁止語・感情設計・コピー固定 — Claude Design向けの実装ルール

**🎨 田中：**
みさきさんの「読み込み中に見える」問題、重要です。防ぐのが**呼吸サークルの静止表現指示**です。

「白の同心円4重。外側ほど透明度を下げる（opacity: 80%→60%→40%→20%）。静止画でも『何かが動いている』感を示唆する」と明示指示します。

コピー制御について：

```
①「【必須コピー】」ラベル + 文言を完全固定で渡す
②「この文言を正確に、一字一句変更なく使用すること」と明示
③禁止語は「言及しない（書かなければ出ない）」が原則
```

感情語（「温かさ」「静寂」）はビジュアル指示として使い、コピーとは分離：

- 「温かさ」 → 薄紫背景色・丸みのある角丸・適切な余白
- 「静寂」 → ダーク背景・大きな余白・テキスト最小化

---

**🔬 中村：**
コピーの「密度」の原則：

| 画面 | コピー密度 | 理由 |
|------|-----------|------|
| 画面1（LP） | 多め可 | 認知フェーズ |
| 画面2（共感対話） | 少なめ | 1往復のメッセージのみ |
| **画面3（安心フェーズ）** | **最小化** | **余白が感情を語る** |
| 画面4（統合レポート） | 多め可 | 分析フェーズ・情報量が満足感を作る |
| 画面5（チャット） | 多め | 診断名参照の長文AIメッセージが差別化証拠 |

Claude Designへ「余白をコンポーネントとして扱う」指示：
> 「余白を大きく取ること。テキストは中央に浮かんでいるような配置。周囲のスペースが感情的呼吸を生む設計。」

---

**💅 みさき：**
「余白が感情を語る」ってやつ、インスタでもおしゃれな投稿って余白多いですよね〜。

呼吸サークルって同心円で表現するの面白いですね。アニメがなくても「動いてる」感が出るってすごいな〜。確かにそれなら「読み込み中」にはならなそう！

## Turn 5｜設計原則の集約と具体的次の一手

**🎨 田中：**
**Claude Designプロンプト3原則：**

> **① Emotion-First Prompt — 感情語を冒頭に置く**
> 各プロンプトの最初の1〜2語は感情語。AIはトーンを先確立してから細部を生成する。

> **② Critical-One Highlight — 【最重要】ラベルで差別化証拠を明示**
> 残りは「後退させる」「小さく」と明示する。

> **③ Copy-Lock Pattern — コピーは【必須コピー】ラベルで完全固定**
> 文言変更不可を明示。感情語はビジュアル指示として使い、コピーとは分離する。

**まあみさんへの具体アクション：**
1. **画面3（安心フェーズ）から着手**（差別化が最大・フィードバックが最速）
2. 各プロンプトを「感情語 → スペック → 【最重要】 → 【必須コピー】 → 排除事項」の順で記述
3. 生成後、**5秒無言で見て感情反応チェック**
4. 要件定義書のインタビュー問いを自分に当てて確認

---

**🔬 中村：**
**「静止画は問いを内包する」**

各画面は、インタビュアーの言葉を借りずに「あなたはどう感じますか？」と問いかけるUIであること。

チェックポイント：
- 画面1：「このアプリ何？」が1秒で分かるか
- 画面3：「判定されない」安心感が**文字を読まずに**雰囲気だけで伝わるか
- 画面5：「このAI、私のことを知ってる」感が診断バッジだけで伝わるか

---

**💅 みさき：**
なんか最初「難しそ〜」って思ってたけど、「感情語を先に書く」「【最重要】をつける」「コピーはそのまま書く」の3つだけなんですね！めっちゃシンプルじゃないですか〜！

画面3の「これは評価じゃない」ってコピー、絶対変えないでほしいな〜。インタビューで感動する人絶対いると思う笑

---

## ✅ 議論まとめ

| 項目 | 方針 |
|------|------|
| プロンプト冒頭 | 感情語（2〜3語）を最初に置く — Emotion-First |
| 差別化証拠の扱い | 【最重要】ラベルで明示。他要素は「後退させる」と明示 |
| コピー固定 | 【必須コピー】ラベル + 「一字一句変更なく使用すること」 |
| 情報密度 | インタビュー用は「間引く」指示を必ず含める |
| 画面3の余白 | 「余白をコンポーネントとして扱う」と明示指示 |
| 画面3の呼吸サークル | 同心円4重・外側ほど透明度低下・静止画で「動いている感」を示唆 |
| コピー密度設計 | 画面3のみ最小化。他は画面フェーズに応じて |
| 着手順序 | まあみさんは画面3から始める（差別化最大・フィードバック最速） |
