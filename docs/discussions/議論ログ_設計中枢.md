## Turn 1｜診断 — 既存資料に「思想の中枢」が無いとどうなるか

**早乙女**：まず現状を観察します。リーンキャンバスv3、Visionary Framework、要件定義書v1.3、言語設計文書v1.0——4つを横断して読みました。「現代の無明」「老子の自知者明」「ハーバード成人発達研究」「自己理解→感情的安心→行動変容」、これらの哲学的キーワードは **散在しています**。重複もしていますが、**それらを生み出した思考の枠組みが、どの文書にも書かれていない**。

**空知**：仏教学の用語で言えば、「教義（ドグマ）はあるが、教学（システマチック・セオロジー）がない」状態ですね。「無明を晴らす」とミッションには書いてあるが、「無明とは何か、なぜそれを晴らすことが解決になるのか」が、論理として一本通っていない。

**みさき**：え待って待って。「ミッションあるのに思想がない」ってどういうこと? 「自分を知って、ラクになる」って書いてあるじゃん? それで足りなくない〜?

**早乙女**：いい質問です。具体例で答えます。たとえば近い将来、「**占い相性診断機能を追加すべきか?**」という議論が必ず起きます。「自分を知ってラクになる」というミッションだけでは、判断できません。なぜなら、占い相性診断も「自分を知る」の一部だから。**判断するには、ミッションの上位にある「方向性原則」が要る**——それが今は不在です。

**空知**：実際、過去にチームは「体癖（taiheki）」を入れるか外すかで揺れて、最終的に外した。その判断は事後正当化されているが、**判断時点で設計中枢に照らせば自動的に答えが出る状態にはなっていない**。これは判断ごとに毎回ゼロから議論することを意味します。

> **🔍 抽象的診断**：COCOSiLの問題は「方向性原則の不在」。
> ミッションは旗印であって判断基準ではない。**判断基準としての設計原則が抽出されていない**ことで、機能の取捨選択ごとに思想が揺らぐリスクを抱えている。

## Turn 2｜構造 — 3つの素材をどう一本の論理に編むか

**空知**：今回扱う3つの素材——「無明」「三毒と方法論」「ハーバード研究」——は、**それぞれ階層が違う**。並列に並べると論理が崩れます。

| 素材 | 階層 | 答えている問い |
|---|---|---|
| 無明（自己への無知） | **世界観** | 何が問題か（Why） |
| 三毒・五蘊・パンチャ構造・止観 | **人間観／方法論** | どう解くか（How） |
| ハーバード成人発達研究 | **価値の確証** | 解いた先に何があるか（So What） |

**早乙女**：戦略フレームに翻訳すると「**Why → How → So What**」の三段論法になります。これは Simon Sinek の Golden Circle とも構造的に整合する。重要なのは、Why から始まること。**「無明」を出発点に置かない設計中枢は、ただの機能仕様書になります**。

**みさき**：あ、なるほど〜。それって「なんで存在するの→どうやって解くの→解いたら何が手に入るの」って3つに分けるってこと? それならめっちゃ整理されてて分かるかも。

**空知**：そう、その3層。みさきさんの口語訳が一番正確です。具体配置を出すと：

- **Why層**：現代人は対人関係で消耗している。原因は「自分も相手も、なぜこうなるかが分からない」という無明。これは2500年前の仏陀の診断と同じ普遍構造。
- **How層**：無明を晴らすには三毒（貪・瞋・痴）の自動連鎖を観察する必要がある。観察の道具立てが五蘊／パンチャ構造（4+1=5）。プロセスは止観の往還運動。
- **So What層**：自己理解は終点ではなく、良い人間関係への入口。ハーバード研究が80年かけて証明した「幸福の決定要因」がそこ。

> **🏗 抽象的軸**：3層構造（Why/How/So What）を骨格にして、各層に哲学素材→現代語訳→プロダクト適用ルール、を並べる。

## Turn 3｜リトマス試験紙化 — 設計中枢が"判断機能"として動くために

**早乙女**：ここからが核心です。設計中枢は「読み物」にしてはいけない。**判断のリトマス試験紙**でなければ意味がない。過去の判断事例を集めてパターン化しましょう。

| 過去の判断 | 結論 | 判断時に効いた問い |
|---|---|---|
| 体癖を入れるか | 外した | 自己理解の解像度を本当に上げるか? |
| SNSシェアは独立か統合か | 独立 | 共感→分析の流れを壊さないか? |
| アクション記録のUI | 「できたこと」形式 | 三毒（特に焦り＝瞋）を煽らないか? |
| 共感→安心→分析→行動の順序 | 非交渉で固定 | 順序を崩したらユーザーが離脱するインタビュー証拠あり |

**空知**：これらに共通するのは「**本質的目的（無明からの解放）に資するか**」が判定基準になっていること。仏教で言う「方便（ほうべん）の判断」です。これを5問に落とし込めれば、誰でも使えるリトマス試験紙になります。

**🧪 5問のリトマス試験紙（試案）**

1. **無明を晴らすか?**（自己理解の解像度を上げるか）
2. **三毒を増幅していないか?**（欲・怒り・無知を煽る方向ではないか）
3. **共感→安心→分析→行動の順序を守れているか?**
4. **小我の強化か、大我への移行を支援するか?**（「反応する自分」の補強ではなく「観てから選ぶ自分」を支援するか）
5. **良い人間関係に寄与するか?**（ハーバード基準）

**みさき**：ちょっと待って〜。これって5個全部○じゃないとダメなの? たとえば「占いガチャ」みたいなライトなエンタメ機能だと、Q1は○でもQ4はバツになりそうじゃん。それで即NGって判断、けっこうキツくない?

**早乙女**：本質を突いてくれました。**Must通過 × Should通過に分けるべき**です。

| 問い | レベル | 理由 |
|---|---|---|
| Q1（無明を晴らすか） | 🔴 Must | プロダクトの存在理由そのもの |
| Q2（三毒を増幅しないか） | 🔴 Must | ここを落とすと害になる |
| Q3（順序を守るか） | 🔴 Must | インタビューで帰納的に検証済み |
| Q4（大我への移行支援） | 🟡 Should | 戦術的に許容範囲あり、ただし明文化必須 |
| Q5（良い人間関係への寄与） | 🟡 Should | 短期施策では遠回りに見えても、長期で必ず接続させる |

**空知**：Must3つを一つでも落とす機能は「方便」ではなく「逆方便」——目的から離れさせる装置になります。Shouldは戦略的判断の余地を残す。

> **🎯 抽象パターン**：判断は単一基準ではなく階層化された問いのセット。「Must必須通過 × Should推奨通過」のレイヤーで、原則の厳密さと現場の柔軟さを両立する。

## Turn 4｜翻訳ルール — 仏教用語と現代語の運用法則

**空知**：用語運用は、思想の鮮度を決めます。具体的な翻訳テーブルを置きましょう。

| 内部用語（仏教） | 対外用語（現代語） | UI/コピーで使う言葉 |
|---|---|---|
| 無明 | 自己への無知・気づきの欠如 | "なぜ自分はこうなのか分からない" |
| 三毒（貪・瞋・痴） | 欲求・怒り・混乱の3パターン | "イライラ／モヤモヤ／焦り" |
| 五蘊・パンチャ構造（4+1=5） | 認識の5要素／4視点の統合 | （UIには出さない、内部設計のみ） |
| 小我（しょうが） | 反応してしまう自分 | "つい〇〇しちゃう私" |
| 大我（仮の自我） | 落ち着いて選べる自分 | "いったん観てみよう" |
| 止観（しかん） | 観察と再起動 | "立ち止まって、見てみる" |
| 業（カルマ）／習気（じっき） | 反応のパターン／クセ | "あなたのいつものパターン" |
| バランスシート（自我の貸借表） | 心の損得勘定 | "なんかモヤッとする時の正体" |

**みさき**：てか「気づきの不足」と「無明」って何が違うの? どっちでもよくない?

**空知**：これは決定的に違います。「気づきの不足」は **知識を足せば解消する一時的状態** を含意します。「無明」は **自己が自己を見えないという構造そのもの** を指す。前者の対処法は「教える」、後者の対処法は「構造を観察させる」。COCOSiLが提供するのは後者です。前者ならググれば足ります。

**みさき**：あ〜！ つまりUIでは「無明」って言葉は使わないけど、**チームで設計してる時には「これは無明を晴らす機能か? それとも知識を足してるだけか?」って問わなきゃダメ**ってこと?

**空知**：完全に正しい。みさきさんの今の翻訳が、設計中枢の運用そのものです。

**早乙女**：ここから抽象的な運用法則を抽出します。

> **🧭 翻訳の運用2法則**
> ① **意味の重みを失わせない**：内部議論で「気づきの不足」と言ってはいけない。「無明」と言うべき。重みが消えると判断が劣化する。
> ② **方向性は一方通行**：チーム内では仏教用語で議論し、外向けには現代語に翻訳する。**逆はやらない**——外で仏教用語を使い、内で現代語のみで議論すると、思想が薄まる典型的失敗パターン。

## Turn 5｜統合 — COCOSiL設計中枢の3原則と次の一手

**早乙女**：ここまでの議論を、**チームに刻める3原則**に集約します。覚えやすく、判断時に唱えやすく、英語のフレーズで記憶のフックを作ります。

### 【COCOSiL設計中枢の3原則】

> **① Dispel, Don't Decorate.（無明を晴らす、装飾しない）**
> 機能は「自己理解の解像度を上げるか?」だけで判断する。エンタメ的な飾り、SNS映え目当ての足し算、目新しさだけの新規性は入れない。

> **② From Reaction to Reflection.（反応から、観察へ）**
> 三毒（欲・怒り・混乱）を増幅する設計を絶対に入れない。すべての体験は「反応してしまう自分（小我）」から「観てから選ぶ自分（大我）」への移行を支援する。

> **③ Self-Knowing for Better-Relating.（自己を知るのは、関わるため）**
> 自己理解は終点ではない。良い人間関係への入口である。ハーバード研究の発見が、すべての機能の最終評価軸。

**空知**：原則は打ち上げただけでは効かない。**今日から動ける具体アクションに着地させる**必要があります。

**🚀 明日からの一手（5項目）**

1. **3原則をSlackチームトピックにピン留め**（毎日視界に入る場所に）
2. **次のスプリントレビューで、未決事項を5問のリトマス試験紙に通す**（占い相性診断・キャラクターカスタマイズなど）
3. **言語設計文書を v1.1 に更新**：設計中枢の翻訳テーブルを取り込む
4. **対外コンテンツの現代語訳指針を1ページにまとめる**：まあみと共有してLP・SNS文言の指針に
5. **月次「思想ドリフト・チェック」**：過去1ヶ月で追加された機能・コピー・プロンプトが3原則から逸脱していないか棚卸し（空知が推奨するレビュープロトコル）

**みさき**：なんか最初は「思想なんて抽象的で意味あるの?」って思ってたんですけど、3原則に落ちると「あ、これがあれば次の機能の判断で迷わないんだな」って分かるわ〜！ 特に **Self-Knowing for Better-Relating** ってフレーズめっちゃ刺さる。MBTIで自分のこと知っただけだと、なんかマウント取りたくなるだけだったし、「関係性のための自己理解」って軸はマジ大事〜。私みたいなユーザーにとって、これが一番の救いになる気がする。
