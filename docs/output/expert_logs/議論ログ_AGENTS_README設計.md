## Turn 1｜現状診断：2つのファイルが抱える「すれ違い」

README は `create-next-app` のデフォルトのまま。npm/yarn/pnpm の4択が残り、pnpm移行済みのプロジェクトと不整合。

AGENTS.mdは「べき論」が書かれているが、Claudeが実際に迷う場面（src/なし、pnpm、process.env禁止）への対処がない。

**診断**: 2ファイルとも「ターゲット読者とファイルの内容がすれ違っている」。READMEは汎用Next.jsの説明をしている。AGENTS.mdは人間向けの設計思想を語っている。

エージェントが犯すミスのタイプ（頻度順）:
1. ディレクトリ構造の誤解（src/ありと思い込む）★★★★★
2. パッケージマネージャの混用（npm/pnpm）★★★★☆
3. 型定義ファイルの場所を間違える ★★★☆☆
4. 環境変数を直接 process.env で読む ★★★☆☆
5. 禁止語彙を使ったUI文言を生成する ★★☆☆☆

## Turn 2｜設計フレームワーク：読者と目的の2軸

| | 人間が読む | エージェントが読む |
|---|---|---|
| **ゴール: 動かす** | README（セットアップ→動作確認） | AGENTS.md（ミス防止パターン） |
| **ゴール: 理解する** | docs/concepts/ | docs/setup/ |

- **READMEの唯一のゴール**: 新メンバーが10分以内に `pnpm dev` を動かせること
- **AGENTS.mdの唯一のゴール**: Claudeが「よくある間違い」を犯さないこと

README必須コンテンツ:
- 🔴 プロジェクト一言説明・セットアップ手順・開発コマンド
- 🟡 アーキテクチャ概要（src/なし、@/*）・環境変数一覧
- 🟢 フィーチャー一覧・デプロイ方法

## Turn 3｜要件確定：「10分テスト」と「ミス防止テスト」

**READMEの10分テスト**:
新メンバーがREADMEだけを読んで10分以内にlocalhost:3000を表示できるか？

**AGENTS.mdのミス防止テスト**（追加基準）:
1. Claude がこの情報なしに犯す間違いのコストは100ドル相当以上か？
2. コードベースを読めば分かることではないか？
3. 他のドキュメントへの参照で十分ではないか？

COCOSiL固有のミス防止リスト:

| ミスの内容 | 修正コスト |
|---|---|
| `src/lib/env.ts` を作ろうとする（src/が存在しない） | 高 |
| `npm install` / `yarn add` を実行する | 中 |
| `process.env.XXX` を直接読む | 中 |
| 「占い結果は…」という文言を生成する | 中 |
| Next.js固有APIを確認せず書く | 高 |
| Supabase型を手書きする | 高 |

## Turn 4｜深掘り：AGENTS.mdは「税金」という法則

> 「AGENTS.mdの1行は、そのプロジェクトで行われる全会話のトークンコストを永久に押し上げる税金だ」

**非対称設計が正しい**:
- AGENTS.md = 最小化・精密化（トークン税を下げる）
- README.md = 親切・完全（人間のUXを最大化する）

追加基準フィルタを適用した結果、新規追加は2セクション（約200トークン）に絞られた。

---

## Turn 5｜設計原則と最終構造

### 3原則

**① Asymmetric Audiences**: AGENTS.mdとREADMEは異なる読者向け。同じ内容を2箇所に書かない。

**② Mistake-First for Agents**: AGENTS.mdに書く価値があるのは「コストの高いミス防止情報」のみ。

**③ 10-Minute Setup for Humans**: READMEの成功指標は「新メンバーが10分以内に動かせるか」。
