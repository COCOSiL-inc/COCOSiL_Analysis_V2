# COCOSiL V2 UI静止画モック要件定義書（モバイル版・インタビュー用）

> **使用目的：** COCOSiLに関する顧客予定者へのインタビューで使用し、製品のイメージを掴んでもらう
> **設計根拠：** `docs/discussions/議論ログ_UI静止画モック要件定義.md`

---

## 1. 概要

### 1.1 インタビュー用モックの設計思想

インタビュー用の静止画モックは「機能説明の資料」ではなく、**「腑落ち体験の感情的旅路を追体験させる刺激材」** として設計する。

COCOSiLのUXシーケンス「共感→安心→分析→行動」が、インタビュー相手の心の中でも同じ順序で動くよう、5画面を設計する。

### 1.2 設計3原則

> **① Emotion First, Feature Second — 感情を先、機能説明は後**
> 各画面の主役は「感情を動かす1要素」。テキスト・視覚要素・CTAはすべてこの原則でフィルタリングする。機能仕様は後から口頭で補完できるが、感情の引き力は補完できない。

> **② Journey in 5 Frames — 5画面で旅路を完結させる**
> LP（何のアプリか）→ 共感対話（AIが話しかけてくれる）→ 安心フェーズ（判定されない）→ 統合レポート（深く分析される）→ チャット（理解された上で対話できる）。この順番はUXシーケンス「共感→安心→分析→行動」そのもの。変えない。

> **③ Proof in Every Screen — 各画面に差別化証拠を1つ**
> インタビュー相手が「他のアプリと何が違うの？」と聞いたとき、各画面が自分で答えられること。口頭説明に頼らない自己証明UIを設計する。

### 1.3 画面選定の理由

| 候補 | 採否 | 理由 |
|------|------|------|
| LP/トップ | ✅ 採用 | 「何のアプリか」の認知入口として必須 |
| オンボーディング共感対話（F1.3） | ✅ 採用 | 「診断前に感情を聞く」という業界初体験を伝える |
| 安心フェーズ（F3.2） | ✅ 採用 | V2最大の差別化要素・「判定されない」体験 |
| 統合レポート（F3.1+F3.3） | ✅ 採用 | 4体系統合の独自価値を視覚化 |
| 共感AIチャット（F4） | ✅ 採用 | 継続使用の核心・「根拠ある共感」を体感 |
| MBTI診断画面（F2.2） | ❌ 除外 | 「選択肢を選ぶ画面」は他社と差別化できない |
| シェアカード（F6） | 🔀 統合 | 統合レポート画面の下部に組み込み |
| アクション記録（F5） | 🔀 統合 | 共感AIチャット画面の下部に導線として組み込み |

### 画面3：安心フェーズ（F3.2）

**担当フェーズ：** 安心（UXシーケンス②）
**差別化証拠：** 「これは評価ではなく、あなたを理解するための地図です」宣言テキスト

> ⭐ **V2最大の差別化画面。インタビューで感情反応を最重点で観察する。**

#### レイアウト

```
[ステータスバー（白テキスト）]
─────────────────────────────
（#1E0B3B〜#5B21B6 暗い紫グラデーション・全画面・白テキスト）


    　　（大きな余白）


    あなたの結果が
    揃いました。

    読む前に、
    ひとつだけ。


         ○            ← 呼吸サークル（白・波紋表現）
       ○   ○
      ○     ○        ← 波紋の静止画表現
       ○   ○
         ○


  ─────────────────────────
  これは評価ではありません。

  あなた自身を理解するための
  地図です。
  ─────────────────────────


  ╔═════════════════════════╗
  ║       読んでみる →       ║  ← 半透明白ボタン・Thumb Zone
  ╚═════════════════════════╝
─────────────────────────────
（ボトムナビなし——没入体験のため）
```

#### 要素定義

| 要素 | 内容 | 優先度 |
|------|------|--------|
| **背景** | `#1E0B3B`〜`#5B21B6` 暗い紫グラデーション・全画面・他画面と明確に異なる | 🔴 必須 |
| **メインコピー（行1）** | 「あなたの結果が揃いました。」 大見出し・白・中央 | 🔴 必須 |
| **メインコピー（行2）** | 「読む前に、ひとつだけ。」 | 🔴 必須 |
| **呼吸サークル** | 白円形＋波紋の静止表現（アニメーション存在を示唆する視覚的演出） | 🔴 必須（差別化証拠） |
| **AIサブテキスト** | 「これは評価ではありません。あなた自身を理解するための地図です。」白テキスト・中央 | 🔴 必須（差別化証拠） |
| **CTAボタン** | 「読んでみる→」半透明白ボタン・Thumb Zone内 | 🔴 必須 |
| **ボトムナビ** | **非表示**（没入体験を壊さないため） | 🔴 必須（排除） |

#### デザイン注意事項
- 「評価」「判定」「スコア」等の語は使わない（language-design準拠）
- 「地図」メタファーを視覚的にも示唆できるなら加点（例：薄く地図の線描）

#### インタビュー問い
> 「この画面を見てどんな気持ちになりますか？」
> **確認する仮説：** 「判定されない安心感」が感情として伝わるか。競合との差別化を自発的に語るか。

## 4. インタビュー活用ガイド

### 4.1 インタビューでの提示順序

```
画面1（LP）→ 画面2（共感対話）→ 画面3（安心）→ 画面4（レポート）→ 画面5（チャット）
```

この順序は**COCOSiLのUXシーケンスと一致している**。変えない。

### 4.2 提示方法の原則

⛔ **「説明してから見せる」は禁止**
✅ **「まず見てもらう → 感想を聞く → 説明で補完する」を厳守**

各画面は「説明なし」で機能するよう設計されている。インタビュアーが事前説明するとモック単体の評価が歪む。

### 4.3 インタビュー問いセット

| 画面 | 主問い | フォローアップ問い | 確認する仮説 |
|------|--------|-----------------|-------------|
| LP/トップ | 「このアプリは何をするサービスだと思いますか？」 | 「どこを見てそう思いましたか？」 | プロダクト理解の到達速度 |
| オンボーディング共感対話 | 「AIにこう聞かれたら、答えたいと思いますか？」 | 「今まで使ったアプリでこういう体験はありましたか？」 | 共感対話への参加意欲 |
| 安心フェーズ | 「この画面を見てどんな気持ちになりますか？」 | 「診断系アプリで結果を見る前ってどんな気持ちになりますか？」 | 「判定されない」安心体験の感情価値 |
| 統合レポート | 「この結果を見て、友達に見せたいと思いますか？」 | 「このボタン（しっくりきたマーカー）を押したいと思いますか？」 | シェア衝動と腑落ち体験の連動 |
| 共感AIチャット | 「自分の診断結果を踏まえてAIと話せるとしたら、使いたいと思いますか？」 | 「普通のAIチャットと何が違うと思いますか？」 | 継続使用意欲・差別化認識 |

### 4.4 最重点観察ポイント

**画面3（安心フェーズ）を最重点で観察する。**

「これは評価ではありません。地図です。」というコピーへの反応が：
- **ポジティブ**（「え、こういう画面見たことない」「安心する」）→ V2差別化の核心が伝わっている ✅
- **無反応 / 分からない** → コピーの見直しまたは視覚表現の強化が必要 ⚠️
