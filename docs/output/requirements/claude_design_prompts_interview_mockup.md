# COCOSiL V2 Claude Designプロンプト集
## モバイルUIモック（インタビュー用）— まあみへ

> **目的：** 顧客予定者インタビュー用の主要5画面UI静止画モック（モバイル版）をClaude Designで生成する
> **設計根拠：** `docs/discussions/議論ログ_UI静止画モック要件定義.md`
> **要件定義：** `docs/output/requirements/ui_mockup_requirements_interview.md`

---

## 使い方ガイド（まあみへ）

### プロンプト3原則（必ず守ること）

> **① Emotion-First Prompt**
> 各プロンプトの冒頭は「感情語」から始まっている。Claude Designはこのトーンを先に確立してから細部を生成する。変更しないこと。

> **② Critical-One Highlight**
> 各プロンプト内の【最重要】タグがついた要素が「差別化証拠」。これが最も目立つように生成されているか確認する。

> **③ Copy-Lock Pattern**
> 【必須コピー】タグの文言は一字一句変更しない。言語設計ルールの遵守のため。

### 着手順序（推奨）

```
画面3（安心フェーズ）→ 画面2（共感対話）→ 画面4（統合レポート）→ 画面5（チャット）→ 画面1（LP）
```

→ 画面3がV2最大の差別化画面。ここで「感じが出ているか」を先に確認する。

### 生成後チェック（各画面共通）

- [ ] 生成後、**5秒間無言で見て**感情反応を確認（言葉より先に感情が動くか）
- [ ] インタビュー問い（各プロンプト末尾）を自分に当てて確認
- [ ] 禁止語（占い・鑑定・運勢・当たる・霊感・霊視）が含まれていないか
- [ ] 主要CTAがThumb Zone（画面下部40%以内）にあるか
- [ ] フレームサイズが375×812pxになっているか

| 項目 | 値 |
|------|-----|

| ブランドカラー（メイン） | `#5B21B6`（紫） |

| フォント | 日本語：Noto Sans JP / 英数：Inter |
| Thumb Zone | 画面下部40%以内に主要CTAを配置 |

## 画面1：LP / トップ

**感情フェーズ：** プロダクト認知（入口）
**差別化証拠：** 「共感→安心→分析→行動」UXシーケンス図

### プロンプト

```
期待と好奇心と信頼感。モバイルUIスクリーン、375×812px、Figmaフレーム。

背景：#5B21B6〜#3B0E8F の紫グラデーション（全画面）。
テキスト：すべて白。フォント：Noto Sans JP（日本語）+ Inter（英数）。

レイアウト（上から下）：

1. 上部中央：「COCOSiL」ロゴテキスト・白・24sp

2. 中央上部（大きな余白の後）：
   大見出し「自分を知って、ラクになる。」白・左寄せ・38sp・太め
   サブコピー「4体系統合 × 共感AI」「なぜ自分はこうなのかが、わかる。」白・左寄せ・17sp・2行

3. 【最重要・差別化証拠】UXシーケンス図：
   「共感 → 安心 → 分析 → 行動」の4ステップを横並びで表示。
   各ステップに白の丸アイコン（直径40px程度）。アイコン下にラベルテキスト（白・小）。
   矢印（→）で各ステップを接続。薄い白の横線または背景で区切る。
   水平に4つ並んだ構成。中央揃え。

4. 下部Thumb Zone内：
   白ボタン（角丸12px）「無料で始める」・#5B21B6テキスト・横幅80%・高さ52px・中央
   テキストリンク「すでにアカウントをお持ちの方はこちら」白・小・中央・ボタン下

余白は大きく取ること。装飾・過剰なグラフィックは排除。
情報はこの6要素のみに絞る。

【必須コピー】
・「自分を知って、ラクになる。」（大見出し）
・「4体系統合 × 共感AI」（サブコピー1行目）
・「なぜ自分はこうなのかが、わかる。」（サブコピー2行目）
・「共感 → 安心 → 分析 → 行動」（UXシーケンス図）
・「無料で始める」（CTAボタン）
・「すでにアカウントをお持ちの方はこちら」（テキストリンク）
```

### インタビュー問い
> 「このアプリは何をするサービスだと思いますか？」
> フォローアップ：「どこを見てそう思いましたか？」
> **確認仮説：** プロダクト理解が30秒以内に到達するか

## 画面2：オンボーディング共感対話（F1.3）

**感情フェーズ：** 共感（UXシーケンス①）
**差別化証拠：** AIが「今日、どんな気持ちで来ましたか？」と話しかけるバブル

### プロンプト

```
温かく自然な会話の空気感。親しみやすさと優しさ。
モバイルUIスクリーン、375×812px、Figmaフレーム。

背景：白（#FFFFFF）。メインテキスト：#1A1A2E。
フォント：Noto Sans JP（日本語）+ Inter（英数）。

レイアウト（上から下）：

1. ヘッダー（白背景）：
   左：「← 戻る」テキスト・グレー・小
   中央：ステップインジケーター「● ○ ○ ○」（紫ドット1つ・グレードット3つ）

2. 上部余白（全体の10%）

3. 左寄せ：紫の円形AIアバターアイコン（直径48px）

4. 【最重要・差別化証拠】AIメッセージバブル：
   背景：#EDE9FE（薄紫）・角丸18px・左寄せ・左上に小さい三角（吹き出し）
   iMessageに近い丸みのある吹き出しスタイル。
   テキスト：「こんにちは。今日、どんな気持ちで来ましたか？」
   #1A1A2E・17sp・行間1.6

5. 中央余白（全体の15%）

6. 選択肢ボタン（縦3列）・各ボタン：
   白背景・#5B21B6の枠線（1.5px）・角丸12px・横幅90%・高さ52px・中央
   テキスト：#5B21B6・16sp
   上から：「ちょっとモヤモヤしてる」「自分をもっと知りたい」「特になんとなく来た」

7. テキスト入力フィールド：
   薄グレー枠・角丸10px・横幅90%・高さ44px
   プレースホルダー：「または自由に入力する」グレー・小

8. Thumb Zone内：
   右寄せ「送る →」ボタン・#5B21B6背景・白テキスト・角丸10px・横幅40%

ボトムナビ（画面最下部）：
「ホーム｜レポート｜チャット｜記録」グレーアイコン付き・4項目均等

【必須コピー】
・「こんにちは。今日、どんな気持ちで来ましたか？」（AIバブル）
・「ちょっとモヤモヤしてる」（選択肢1）
・「自分をもっと知りたい」（選択肢2）
・「特になんとなく来た」（選択肢3）
・「または自由に入力する」（プレースホルダー）
・「送る →」（CTAボタン）
```

### インタビュー問い
> 「AIにこう聞かれたら、答えたいと思いますか？」
> フォローアップ：「今まで使ったアプリでこういう体験はありましたか？」
> **確認仮説：** 診断前の感情確認対話への参加意欲があるか

## 画面3：安心フェーズ（F3.2）⭐ 最重点

**感情フェーズ：** 安心（UXシーケンス②）
**差別化証拠：** 「これは評価ではありません。地図です。」宣言テキスト

> ⚠️ **まあみへ：この画面を最初に作る。インタビューで最も感情反応を観察する画面。**

### プロンプト

```
静寂と受容の空気感。深呼吸のような間。何も急かさない安心感。
モバイルUIスクリーン、375×812px、Figmaフレーム。

背景：#1E0B3B（上）〜#5B21B6（下）の暗い紫グラデーション（全画面）。
テキスト：すべて白（#FFFFFF）。他のUIカラーは使わない。
フォント：Noto Sans JP（日本語）+ Inter（英数）。

レイアウト（上から下）：

1. ステータスバー：白テキスト（薄く）

2. 大きな余白（全体の22%）

3. メインコピー1：「あなたの結果が揃いました。」
   白・中央揃え・28sp・太め（Bold）

4. 小余白（8px）

5. メインコピー2：「読む前に、ひとつだけ。」
   白・中央揃え・22sp・標準（Regular）・不透明度85%

6. 余白（全体の12%）

7. 【最重要・差別化証拠】呼吸サークル（中央配置）：
   白の同心円4重。最内円：直径60px・opacity 100%。
   2番目：直径100px・opacity 75%。
   3番目：直径145px・opacity 45%。
   最外円：直径195px・opacity 20%。
   各円は枠線（stroke）のみ・塗りなし。線幅1.5〜2px。
   静止画でも「息を吐いている」「何かが広がっている」感を表現すること。

8. 余白（全体の10%）

9. 【最重要・差別化証拠】AIサブテキスト：
   「これは評価ではありません。」白・中央揃え・16sp・標準
   （1行空けて）
   「あなた自身を理解するための地図です。」白・中央揃え・16sp・標準
   テキストの下に薄い横線（白・opacity 20%）

10. 余白（全体の8%）

11. Thumb Zone内・中央：
    「読んでみる →」ボタン。
    背景：白・opacity 15%（半透明）。枠線：白・1px。
    テキスト：白・16sp。角丸28px。横幅70%・高さ52px。

ボトムナビ：なし（没入体験のため完全排除）。
装飾・アイコン・追加テキスト・ラベルは一切配置しない。
余白そのものが感情的な「間」を作る設計。情報密度は全5画面の中で最低にすること。

【必須コピー】
・「あなたの結果が揃いました。」
・「読む前に、ひとつだけ。」
・「これは評価ではありません。」
・「あなた自身を理解するための地図です。」
・「読んでみる →」
```

### インタビュー問い
> 「この画面を見てどんな気持ちになりますか？」
> フォローアップ：「診断系アプリで結果を見る前ってどんな気持ちになりますか？」
> **確認仮説：** 「判定されない」安心体験が感情として伝わるか。競合との差別化を自発的に語るか。

## 画面4：統合レポート（F3.1 + F3.3 + F6組み込み）

**感情フェーズ：** 分析（UXシーケンス③）
**差別化証拠：** 4体系バッジの視覚的共存（星座×動物×六星×MBTI）× 統合タイプ名

### プロンプト

```
知的満足と腑落ちの空気感。「これ、私のことだ」という発見の感覚。
モバイルUIスクリーン、375×812px、Figmaフレーム。

背景：白（#FFFFFF）。アクセントカラー：#5B21B6（紫）。
薄紫バブル背景：#EDE9FE。テキスト：#1A1A2E。
フォント：Noto Sans JP（日本語）+ Inter（英数）。

レイアウト（上から下）：

1. ヘッダー（白背景）：
   左：「COCOSiL」テキスト・#5B21B6・16sp
   右：「⋮」（縦三点）メニューアイコン・グレー

2. 【最重要・差別化証拠①】4体系バッジ横並び：
   #EDE9FE背景の横帯（padding 10px）。4つのバッジを均等配置：
   各バッジ：アイコン（絵文字またはシンプルアイコン）＋テキスト
   「★ 乙女座」「🦌 こじか」「⭐ 土星人(+)」「🧠 INFJ」
   バッジ形式：角丸8px・#EDE9FE背景・薄紫border・12sp・横並び

3. 区切り線（薄グレー・1px）

4. 【最重要・差別化証拠②】統合タイプ名：
   「夜行性の内省的旅人タイプ」
   #5B21B6・左寄せ・30sp・太め（Bold）・上下16pxの余白

5. セクション見出し：「■ あなたの強みと傾向」
   #1A1A2E・左寄せ・15sp・太め・下に薄い区切り線

6. 本文テキスト（2〜3行分のみ。スクロール示唆あり）：
   「物事を深く考え、表面上の関係より本質的なつながりを求めます。
   職場での孤立感は、あなたが感じていることの…」
   #1A1A2E・15sp・行間1.8
   テキスト末尾：「▼ もっと見る」#5B21B6・小・右寄せ

7. しっくりきたマーカーボタン（セクション右端または本文下）：
   背景：#EDE9FE・角丸20px・padding 8px 16px
   テキスト：「これ自分だ ✓」#5B21B6・14sp

8. 区切り線（薄グレー）

9. シェアカードプレビュー（縮小カード画像）：
   角丸12px・薄い影・横幅90%・高さ90px程度
   カード内：「夜行性の内省的旅人タイプ」と4体系バッジを縮小表示

10. Thumb Zone内：
    「友達にシェアする」ボタン・#5B21B6背景・白テキスト・角丸12px・横幅85%・高さ50px・中央

ボトムナビ（画面最下部）：
「ホーム｜レポート｜チャット｜記録」グレーアイコン付き・4項目均等

【必須コピー】
・「夜行性の内省的旅人タイプ」（統合タイプ名）
・「★ 乙女座」「🦌 こじか」「⭐ 土星人(+)」「🧠 INFJ」（4体系バッジ）
・「■ あなたの強みと傾向」（セクション見出し）
・「これ自分だ ✓」（しっくりきたマーカー）
・「友達にシェアする」（CTAボタン）

禁止事項：「占い」「運勢」「鑑定」「霊」等の語はすべて使わない。
「パーソナリティ」「性格傾向」「特性」のみ使用可。
```

### インタビュー問い
> 「この結果を見て、友達に見せたいと思いますか？理由は？」
> フォローアップ：「このボタン（しっくりきたマーカー）を押したいと思いますか？」
> **確認仮説：** しっくりきたマーカー→シェアの衝動が連動するか。4体系統合の独自価値を認識するか。

## 画面5：共感AIチャット（F4）

**感情フェーズ：** 共感→行動（UXシーケンス①+④）
**差別化証拠：** 診断バッジ表示 + 「Phase 2 / 5：共感」フェーズインジケーターの共存

### プロンプト

```
理解されている安心感と対話の温かさ。「このAI、私のことを知っている」という感覚。
モバイルUIスクリーン、375×812px、Figmaフレーム。

背景：白（#FFFFFF）。アクセントカラー：#5B21B6（紫）。
薄紫：#EDE9FE。テキスト：#1A1A2E。グレー：#F0F0F0。
フォント：Noto Sans JP（日本語）+ Inter（英数）。

レイアウト（上から下）：

1. ヘッダー（白背景）：
   左：「← 戻る」グレー・小
   中央：「共感AIチャット」#1A1A2E・16sp・太め
   右：「⋮」メニューアイコン・グレー

2. 【最重要・差別化証拠①】フェーズインジケーターバー：
   #EDE9FE背景の横帯（padding 10px 16px）。
   左：プログレスドット「● ● ● ● ○」（4つ紫・1つグレー）・横並び
   右：「Phase 2 / 5：共感」テキスト・#5B21B6・13sp

3. 【最重要・差別化証拠②】診断バッジ（小・横並び）：
   白背景・#EDE9FEで各バッジを区切り：
   「★乙女座」「🦌こじか」「⭐土星人(+)」「🧠INFJ」
   各バッジ：角丸6px・#EDE9FE背景・11sp・横並び
   （「この診断データが会話に使われている」ことを視覚的に証明する）

4. 区切り線（薄グレー）

5. チャット履歴エリア（スクロール可能・中央付近）：

   右寄せ・ユーザーバブル：
   「職場のチームメンバーと、また同じパターンに…」
   #F0F0F0（グレー）背景・角丸16px（右上は小さめ）・右寄せ・14sp

   左寄せ・AIバブル：
   #EDE9FE（薄紫）背景・角丸16px（左上は小さめ）・左寄せ・14sp・行間1.7
   iMessageスタイルの吹き出し。左上に小さい三角。

6. 入力エリア（Thumb Zone内）：
   薄グレー枠のテキストフィールド・角丸22px・横幅75%
   プレースホルダー：「メッセージを入力」グレー・小

7. 「これをやってみる → 記録」ボタン（テキストフィールド上部）：
   #EDE9FE背景・#5B21B6テキスト・角丸8px・小

8. 「送る →」ボタン（テキストフィールド右隣）：
   #5B21B6背景・白テキスト・円形または角丸。

ボトムナビ（画面最下部）：
「ホーム｜レポート｜チャット｜記録」グレーアイコン付き・「チャット」がアクティブ状態（#5B21B6）

【必須コピー（AIバブル内）】
「そのモヤモヤ、ずっと一人で抱えてきたんですね。
乙女座の分析志向と、こじかタイプの共感しやすさが組み合わさると、
人の感情を感じ取りすぎてしまうことが…」

【必須コピー（その他）】
・「Phase 2 / 5：共感」（フェーズインジケーター）
・「★乙女座」「🦌こじか」「⭐土星人(+)」「🧠INFJ」（診断バッジ）
・「これをやってみる → 記録」（アクション記録導線）
・「送る →」（送信ボタン）

重要：AIバブルのメッセージは必ず4体系の診断名（「乙女座」「こじか」等）を本文中に含む。
これが「普通のAIチャットと違う」ことの視覚的証拠。
```

### インタビュー問い
> 「自分の診断結果を踏まえてAIと話せるとしたら、使いたいと思いますか？」
> フォローアップ：「普通のAIチャットと何が違うと思いますか？」
> **確認仮説：** 継続使用意欲があるか。差別化を自発的に語るか。
