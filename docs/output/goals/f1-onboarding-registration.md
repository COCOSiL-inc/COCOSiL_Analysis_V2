# Goal: F1 オンボーディング・登録

## Vision

**問題**: 診断系プロダクトを開いた瞬間、ユーザーはいきなり登録フォームや診断質問に投げ込まれ、「自分のためのプロダクトではない」と感じて離脱する。共感の入口がないまま機能に触れさせると、PMF基準（7日内再訪30%）の前段階で消失する。F1 は「共感フェーズに着地するまでを30秒で成立させる」ことで、この初手離脱を防ぐ。

**ユーザー**: 25〜35歳の若手社会人。職場・恋愛・家族との関係で「なぜ自分はこう反応してしまうのか」を言語化したいが、「占い」「鑑定」といったスピリチュアル文脈は信頼できず避けてきた層。SNSで自己理解コンテンツに触れる機会はあるが、診断疲れもしている。

**代替手段**:
- 無料の MBTI / 16Personalities などの単体診断を渡り歩く（毎回 type だけ出て、関係性の文脈に翻訳されない）
- SNS（X / Threads）の自己理解系コンテンツを断片的に消費する
- 友人・パートナーへの愚痴（その場で発散するが、パターン認識は残らない）
- 「占い」系サービスは信頼できず能動的には選ばない（語彙拒否）

いずれも「共感の入口」と「4体系統合の根拠ある自己理解」を同時に提供しないため、繰り返し消耗が続く。

**6ヶ月後の成功像**: LP 到達ユーザーの過半数が30秒以内にサインアップ＋生年月日入力を完了し、ウェルカム対話（F1.3）で「自分のために作られたプロダクトだ」と感じて F2（性格分析）へ自然に流れている — つまり「共感→安心→分析→行動」の最初のジャンプが摩擦なく成立している状態。

## Outcome

**何が変わるか**: LP 到達ユーザーのうち、サインアップ + 生年月日入力 + F1.3 ウェルカム対話の1往復まで を完走する割合が大きく改善する。具体的には:

- **F1 完走率**（LP → F1.3完了）が新規ユーザーの **40% 以上**
- そのうち **80% 以上が30秒以内** に完走
- F1 完走者の **70% 以上が F2（性格分析）に遷移**

**測定方法**:

定量（PostHog ファネル）:
- イベント: `lp_viewed` → `signup_completed` → `birth_date_submitted` → `welcome_chat_replied`（F1.3の1往復完了）→ `f2_entered`
- 各ステップの転換率と所要時間（中央値・90パーセンタイル）を計測
- 30秒以内完了率は `signup_completed` から `welcome_chat_replied` までの経過時間で算出

定性（補完）:
- F1.3 ウェルカム対話後の任意マイクロアンケート（1問: 「このプロダクトは自分のためのものだと感じた？ Yes / Not yet」）
- ユーザーインタビュー（月1〜2件、PMF確認用）で F1 体験の言語化を収集

**期限**: F1 ローンチ後 **30日** で O-1 の閾値を初回計測し達成確認、その後は継続的にモニタリング。ローンチ後 30日 → 90日のあいだに PMF基準（7日内再訪 30%）と突き合わせ、F1 のファネル数値と再訪率の相関を見て調整を回す。

**最低ライン**: 以下のいずれかを下回ったら F1 は「壊れている」と判断する。

- **F1 完走率 25% 未満**（入口が機能していない／LPコピー or 哲学提示が刺さっていない）
- **30秒以内完了率 50% 未満**（フォーム摩擦 or Clerk認証の体感が重い）
- **F1 → F2 遷移率 50% 未満**（共感→分析のジャンプが成立していない／F1.3対話が共感を生んでいない）

いずれか1つでも該当 → ローンチ後 **14日** で原因分析と修正方針確定。

## Eval criteria

- [ ] **E-1-a**: ユーザーが LP → Clerkサインアップ → 生年月日入力 → F1.3 ウェルカム対話（AI 1往復）→ F2 ページ遷移、をブロッカーなしで完走できる（正常系・新規ユーザー）
- [ ] **E-1-b**: F1.3 ウェルカム対話の AI 応答に禁止語（占い／鑑定／運勢／占い師／当たる／霊感／霊視）が含まれない
- [ ] **E-1-c**: 生年月日が未来日付・1900年以前・無効日付の場合、Zodバリデーションでフォーム送信前にブロックされる（インラインエラー表示）
- [ ] **E-2 失敗ケース防止**: F1.3 が「共感の入口」になっていない（診断口調 / 占い的トーン / 三毒増幅 / 定型1パターン）が起きない

**確認方法**:

| 基準 | 確認方法 | 担当 |
|---|---|---|
| E-1-a（完走パス） | プレビューURL手動完走 → 将来 Playwright E2E（Sprint 3以降） | えんまさ（手動）→ 将来自動 |
| E-1-b（禁止語ゼロ） | `lib/prompts/__tests__/welcome-chat.test.ts` の Vitest unit（`expect(prompt).not.toContain('占い')` 等） | ヒラメ（CI） |
| E-1-c（Zod境界値） | `lib/diagnostics/__tests__/birth-date.test.ts` の Vitest unit（未来日付・1900年以前・2/29） | ヒラメ（CI） |
| E-2（共感入口の質） | Gate 2: AI応答サンプル3件（before/after）添付 → えんまさレビュー | えんまさ |
| E-2（中長期） | promptfoo eval（設計中枢5問・三毒増幅判定）Sprint 3以降 | ヒラメ＋えんまさ |

ローンチ時点では「E-1-b/c は自動 + E-1-a/E-2 はえんまさ手動」、ローンチ後に E2E と promptfoo を順次追加。

**劣化検出**:

| 層 | sensor | タイミング |
|---|---|---|
| コード劣化 | Vitest regression（禁止語チェック・Zod境界値）を CI gating で常時実行 | F1 実装時から（即時） |
| 数値劣化 | PostHog ファネル + O-4 閾値割れ時の Slack/メール subscription | F1 ローンチ時 |
| 意味劣化 | promptfoo eval（設計中枢5問・三毒増幅判定・禁止語）を週1自動実行 | Sprint 3 以降 |

ローンチ時点ではコード層 + 数値層のみ。意味層は Sprint 3 で promptfoo を導入してから。それまでは F1.3 のAI応答ログを週1でえんまさが目視レビュー（5件サンプリング）で代替。
