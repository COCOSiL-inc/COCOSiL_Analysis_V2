# deepresearch プロンプト: OpenAI gpt-image-2 を構造化レポート生成 API として活用する方法

> 作成日: 2026-05-07  
> 目的: Gamma API（月額約2万円）の代替として gpt-image-2 API でのレポート生成実現可能性を評価するための調査プロンプト

---

## Research Request: OpenAI gpt-image-2 を構造化レポート生成 API として活用する方法

### 背景

- 現在 Gamma API（月額約 2 万円）を使い、4体系性格分析（MBTI・動物占い・六星占術・星座）の統合レポートを画像として生成している
- OpenAI が 2026年4月末にリリースした gpt-image-2 は、テキスト描画精度が高く、デザイン性もある
- Gamma API の代替として gpt-image-2 API 経由でレポート生成を行う実現可能性を評価したい

---

### 調査してほしいこと（優先度順）

#### 1. gpt-image-2 API の基本仕様と制約

- REST API エンドポイント・リクエスト形式（prompt, size, quality, n など）
- 出力フォーマット（PNG/WebP/JPEG, base64 or URL）
- 1リクエストあたりの最大トークン数・プロンプト文字数制限
- 1画像あたりのコスト（quality=low/medium/high 別、1024×1024 / 1792×1024 / 1024×1792）

#### 2. テキスト描画精度の検証状況

- 日本語テキストの文字崩れ発生率に関する公式・コミュニティの報告
- 長文（500字以上）を含む複数セクションレイアウトでの挙動
- フォント指定・スタイル制御の可否

#### 3. 複数ページ／複数セクションレポートの生成パターン

- 1プロンプトで複数セクション（見出し・本文・アイコン）を1画像に収める手法
- 複数画像を順番に生成してPDFや連続画像に合成するパターン
- プロンプトエンジニアリングのベストプラクティス（レイアウト指示、色・フォントサイズ指定）

#### 4. Gamma API との機能・コスト比較

- Gamma API: レート制限・1レポートあたりのコスト・生成時間
- gpt-image-2 API: 同条件での推定コスト・生成時間・品質比較
- MAU 1,000人規模でのランニングコスト試算（月30レポート/ユーザー想定）

#### 5. Next.js (App Router) からの組み込み方法

- サーバーサイド（API Route / Server Action）から OpenAI SDK で呼び出す実装パターン
- ストリーミング対応の有無
- タイムアウト・リトライ設計（Vercel の 60秒制限への対応）
- Supabase Storage への画像保存フロー

#### 6. 代替アーキテクチャの選択肢

- gpt-image-2 単体 vs gpt-image-2 + HTML→画像変換（Puppeteer/Playwright）の比較
- HTMLテンプレート → Vercel OG / @vercel/og でのレポート画像生成の実現可能性
- 上記の中で「文字崩れなし・日本語対応・低コスト・Next.js 組み込みやすさ」の観点でのランキング

---

### 期待する出力形式

- 各項目に対して「現時点での最新情報（2025年5月時点）」を明記
- コスト比較は表形式で
- 実装パターンはコードスニペット付きで
- 「Gamma APIを捨てて gpt-image-2 に移行すべきか」の結論と根拠を最後にまとめる
